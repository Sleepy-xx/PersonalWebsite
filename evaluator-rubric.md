# PersonalWebsite 评估标准

本评估标准用于在功能调整、内容更新、文档维护或部署前后检查当前个人网站是否符合项目定位。评分不要求一次性重构，只用于识别风险、确认验收证据和记录待确认事项。

## 评分方式

| 分数 | 含义 |
| --- | --- |
| 2 | 符合当前项目目标，已有可观察证据或可通过仓库文件确认。 |
| 1 | 基本可用，但存在内容、体验、文档或验证缺口。 |
| 0 | 明显不符合当前项目，或无法支持公开个人网站使用。 |
| 待确认 | 仓库文件不足以判断，需要本地运行、线上访问或人工核对。 |

| 类别 | 检查问题 | 分数 | 备注 |
| --- | --- | --- | --- |

| 代码可维护性 | 页面、组件、MDX 内容和工具函数是否职责清楚，后续维护者能定位内容来源？ |  | 当前组件拆分较清楚；`src/app/page.js` 与 `page.jsx` 并存待确认。 |
| 文档完整性 | README、ARCHITECTURE、feature list、质量文档和初始化说明是否反映当前项目状态？ |  | 本文档和 `quality-document.md` 应随项目状态持续更新。 |
| 部署可验证性 | GitHub Pages 静态部署流程是否清楚，构建产物和验证命令是否可追踪？ |  | 当前发现 `.github/workflows/deploy.yml`；实际部署结果待确认。 |

## 验收结论

- 接受：关键页面、内容、构建或部署验证均有证据，且没有阻塞公开访问的问题。
- 需修订：主要功能可用，但存在内容准确性、响应式、可访问性、SEO 或文档缺口。
- 阻塞：构建失败、主导航不可用、项目内容无法展示、部署流程不可验证，或存在明显误导性内容。
